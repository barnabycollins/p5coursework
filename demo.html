Perlin Noise

Based on this sketch by Tony R. GitHub repo here.

To update the parameters of the implementation of Perlin Noise on this page, simply type (or select) your desired values in the form below, then click one of the buttons in the bottom right to update the sketch.

The 'update' button will update the parameters instantly without restarting, where the 'restart' button will update them and restart the sketch. Refresh the page to reset the sketch to default values.

Seed

The seed is used to generate the Perlin Noise that produces the patterns you see. A different seed will result in different patterns. NB: the seed needs to be a number.

Number of particles

This decides how many particles should be alive at once. A larger number may produce more pretty lines, but will also run slower.

Mode

0
1

Mode 0 makes all particles spawn at the top by default, where mode 1 will allow particles to spawn anywhere. Mode 0 produces a nice 'cascade' effect from the top, but mode 1 makes nice 'contours' across the whole canvas.

Particle Life

Min
Max

These values determine how long each particle should spend travelling around before it dies and respawns somewhere else. The life of each particle is generated within these bounds.

Noise Scale

The noise scale gives how large the Perlin Noise canvas should be relative to the pixels on your screen. A smaller value will result in more intricate patterns, and a larger value in larger, less detailed patterns.

Simulation Speed

The speed expresses how fast the particles should move around. Very high speeds may cause the trails to look less consistent.

Inner Padding

Vertical
pixels
Horizontal
pixels

The extra space to leave around the inside edge of the canvas in pixels.

Colours

Unfortunately the background colour cannot be changed because the temporal fading (making older trails fade away) relies on being able to continuously darken the whole canvas, meaning any background we try to set would be changed to black in a few seconds.

Default

The colour to use for non-coloured particles.

Left

The colour to use for coloured particles moving left.

Right

The colour to use for coloured particles moving right.
